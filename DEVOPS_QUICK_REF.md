# DevOps Setup - Quick Reference

## 🚀 Quick Start

```bash
# 1. Navigate to terraform directory
cd terraform/

# 2. Initialize
terraform init

# 3. Plan (review changes)
terraform plan

# 4. Deploy to AWS
terraform apply

# 5. Save outputs (EC2 IP, SSH command, URLs)
# These appear after terraform apply completes
```

---

## 📋 What Gets Created

- **VPC** - Virtual Private Cloud in ap-south-1
- **EC2 t2.micro** - Ubuntu 22.04 instance (FREE)
- **Security Group** - Opens ports 22, 80, 443, 4172, 7999, 5432
- **SSH Key Pair** - Saved as `.ssh/user-management-app-key.pem`
- **Elastic IP** - Static public IP address (FREE while instance running)

---

## 🔑 GitHub Secrets Required

Add these to GitHub repo → Settings → Secrets and variables → Actions:

| Secret Name | Value | Where to Find |
|------------|-------|---------------|
| `EC2_SSH_PRIVATE_KEY` | Contents of `.ssh/user-management-app-key.pem` | Created by Terraform |
| `EC2_PUBLIC_IP` | Public IP from Terraform outputs | Terraform apply output |

## 🔄 Deployment Flow

```
Code push to main
       ↓
GitHub Actions triggered
       ↓
SSH to EC2 instance
       ↓
Git pull latest code
       ↓
Docker compose build (fresh images)
       ↓
Docker compose up -d (start containers)
       ↓
Wait for PostgreSQL ready
       ↓
Wait for Backend API ready
       ↓
Wait for Frontend ready
       ↓
Run API tests
       ↓
✅ Deployment complete or ❌ Failed
```

## 🛠️ Common Commands

### Terraform

```bash
# Plan changes (no apply)
terraform plan

# Deploy infrastructure
terraform apply

# Destroy all resources
terraform destroy

# View outputs
terraform output

# Get specific output
terraform output ec2_public_ip
```

### On EC2 (via SSH)

```bash
# View running containers
docker compose ps

# See container logs
docker compose logs -f

# Specific service logs
docker compose logs server
docker compose logs client
docker compose logs postgres

# Restart services
docker compose restart

# Manual rebuild and start
docker compose down
docker compose build --no-cache
docker compose up -d
```

## 🔍 Troubleshooting

| Issue | Solution |
|-------|----------|
| Terraform plan fails | Check AWS credentials: `aws configure` |
| SSH key not found | Key is in `terraform/.ssh/user-management-app-key.pem` |
| Cannot SSH to EC2 | EC2 still initializing, wait 2-3 minutes |
| GitHub Actions fails | Check GitHub secrets are added correctly |
| Docker not found on EC2 | CloudInit script still running, wait 2-3 minutes |
| Ports already in use | SSH to EC2 and run `docker compose down` |
| Database not seeding | Check server logs: `docker compose logs server` |

## 🔐 Security Notes

- SSH key (`.pem`) is sensitive — never commit to Git
- `.gitignore` already excludes it
- GitHub secret stores key securely
- Security group allows all IPs to ports (4172, 7999) — restrict in production
- SSH runs as `ec2-user` (non-root) — safe

## 📝 Environment Variables

App uses `.env` from local repo (same credentials as local):

```
POSTGRES_USER=abinash_naik
POSTGRES_PASSWORD=abinash.naik
POSTGRES_DB=docker_compose_test
```

To override on production, modify `docker-compose.yml` or add `.env.prod` to EC2.

## 🗑️ Cleanup

When free tier expires or you want to stop:

```bash
cd terraform/

# Option 1: Stop instance (pause billing)
aws ec2 stop-instances --instance-ids <id> --region ap-south-1

# Option 2: Delete everything (recommended)
terraform destroy
# Type 'yes' to confirm
```
